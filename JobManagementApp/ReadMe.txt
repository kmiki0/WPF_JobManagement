# 実装予定の機能

## メインウィンドウ
	- ジョブ実行方法がCMDの場合、「運用処理管理R更新」のダイアログを出さずに、CMDコマンドをTextBoxに表示するようなウィンドウを表示する。
	- ジョブ追加のボタン追加
		→　追加後、リスト更新する（開いた状態を維持したまま）	
	- 他ウィンドウが開いている状態では、閉じられないようにする？連鎖的に全部閉じる？

## ジョブ設定画面
	- 文字数上限のカット　追加
	- 閉じるボタン、押下時に編集しているかチェックして、編集していたら登録を促す
	- 削除ボタンで、確認ダイヤログ後に削除するようにする。→その後、メインウィンドウに戻す
	- 更新ボタンのチェックで、ジョブ実行方法がCMDの場合、CMDコマンドを必須項目として判定する。

## ログ画面
	- 常時更新されるログがある場合、更新タイミングを手動にするボタン　追加
	- 取得するのは、本日日付のもの
	- リスト更新時、画面新しくするときに、全面表示とCallbackのイベントを付与する（初期表示時と同じ対応をする）
